我们为什么要学习排序算法呢？

排序就是将一组对象按照某种逻辑顺序重新排列的过程，在平时编程中虽然往往不需要自己写排序算法（编程语言往往都封装了优秀的排序算法），但是学习排序算法还是十分有意义的：

  1. 对排序算法的分析有助于理解其他比较算法性能的方法
  2. 类似的技术可用于解决其他的问题（举一反三）
  3. 排序往往是我们解决其他问题的第一步骤

一些预备知识：

  1. 需要注意的是，默认的排序是指将数组按升序排序
  2. 主键：可以这样理解，一个对象可能有多个属性，但我们只看重要的那个。比如一个学生有三门成绩（语数英），但我们现在想通过数学成绩排序，那么数学成绩可以理解为主键。也就是说主键是我们要比较的东西
  3. 在Java中，对象的比较可以通过实现Comparable接口或Comparator来实现，在这个文章中默认比较的对象都实现了Comparable，并使用Comparable数组的方式来代表（这个涉及到接口类型变量的知识，可以简单理解为Comparable a = b代表着b是一个实现了Comparable接口的对象，其可进行比较）。
  4. 算法的稳定性是指主键相同的数组元素在排序前后它们的相对次序不会发生变化

1. 冒泡排序

1. 原理

冒泡排序是很简单的排序方法，它的核心思想就是“落井下石”，意思就是依次将主键最大（对于单纯的数字而言就是就数值更大的意思）的元素沉到最后，而较小的元素会不断上浮，就像石头砸进水中沉底而气泡漂浮一样，这也是冒泡排序名字的由来（个人理解）。另外冒泡排序注重与元素本身（看到选择排序时你会注意到不同）

冒泡排序通过不断交换不符合顺序的相邻元素来实现排序，其每次均会通过比较找到最大的元素并将其落到最后一位。当最后一个元素落到数组头时，排序自然而然就成功了。

2. 实现

冒泡排序伪代码：

BubbleSort：

Loop (i: From 0 to N-1):

​ Loop (j: From 0 to N-i-1):

​ if arr[j+1] < arr[j]:

​ 交换arr[j+1]与arr[j]

==疑问：==

当然你会发现一个问题，外层循环的次数是固定的（没错，就是N次），如果在N次之前数组已经有序了呢？那我们岂不是浪费了大量的时间？（可以极端地假设原数组就是有序的，那么我们其实在接下来的N次大循环中什么也没做）我们可以设置一个额外的布尔变量flag来记录是否发生了交换（如果发生了交换那么flag保持为false，如果未发生交换flag就变为true），也就是我们在大循环上加入一个判断，如果flag为true就退出循环，伪代码如下:

BubbleSort：

bool flag =false

Loop (i: From 0 To N-1):

​ if !flag:

​ Loop (j: From 0 To N-i-1):

​ if arr[j+1] < arr[j]:

​ 交换arr[j+1]与arr[j]

​ flag = true

​ else: 直接退出

//但是这样改进之后可能会为我们之后的性能分析带来一定困难

下面给出java语言版本的实现并给出基本测试

冒泡排序Java实现

package com.qisir.pku.algorithm.sort;

import java.util.Random;

public class Bubble {
	//初级冒泡排序实现
	public static void sort(Comparable[] arr) {
		for (int i = 0; i < arr.length; i++) {
			for (int j = 0; j < arr.length-i-1; j++)
				if (less(arr[j+1], arr[j]))
					exch(arr, j+1, j);
		}
	}
	
	//优化的冒泡排序（优化后性能还是很低）
	//i所在的外层循环一定会完全走完，但是有可能在i次之前就已排序完毕（标志是i-1次未发生任何交换）
	//通过boolean变量flag来判断是否存在交换，若已不存在，就退出，降低时间复杂度
	public static void advancedSort(Comparable[] arr) {
		boolean flag = false;
		for (int i = 0; i < arr.length-1; i++) {
			flag = false;
			for (int j = 0; j < arr.length-i-1; j++) {
				if (less(arr[j+1], arr[j])) {
					exch(arr, j+1, j);
					flag = true;
				}
			}
			if (!flag) return; 
	    }
	}
	
	// 比较两可比较对象，返回v是否小于w
	private static boolean less(Comparable v, Comparable w) {
		return v.compareTo(w) < 0;
	}
	
	//交换被排序数组中两个元素
	private static void exch(Comparable[] arr, int i, int j) {
		Comparable t = arr[i];
		arr[i] = arr[j];
		arr[j] = t;
	}
	
	//输出数组内容
	public static void show(Comparable[] arr) {
		for (int i = 0; i < arr.length; i++)
			System.out.print(arr[i]);
		System.out.println();
	}
	
	//检查数组是否已经按从小到大的顺序排列
	public static boolean isSorted(Comparable[] arr) {
		for (int i = 9; i < arr.length; i++)
			if (less(arr[i], arr[i-1])) return false;
		return true;
	}
	
    //简单地测试冒泡排序
	public static void main(String[] args) {
		Random random = new Random();
		Comparable[] arr = new Comparable[10];
		for (int i = 0; i < arr.length; i++)
			arr[i]  = random.nextInt(10);
		show(arr);
		sort(arr);
		show(arr);
	}
}

2. 选择排序

1. 原理

选择排序的核心思想是”能者优先“， 也就是说我们每次从余下的全部元素中选出最小的并放在合适的位置上。选在排序注重于位置，我们每次都是为一个位置选择适合该位置的元素。

选择排序通过为第i个位置选出第i大的元素完成排序。

2.实现

选择排序伪代码

SelectionSort:

Loop (i: From 0 To N-1):

​ set min = i

​ Loop (j: From i DownTo N-1):

​ if arr[j] > arr[min]:

​ min = j

​ 交换arr[i]与arr[min]

当然有同学会意识到如果要找的元素就在恰当位置上那么我们会做原地交换，这样可能浪费了一定的性能。可以通过判断min是否与i相等，如果相等就不需要交换了。这个想法确实会有一定帮助，但是这样会导致对选择排序性能的分析变得更困难（因为性能与输入有关联了），而且这种改动可能仅仅优化了常数级别（也就是说如果原算法性能不够了，这种优化也并不会起作用，我们会选择更高级的排序算法）

像C.A.R. Hoare（快速排序发明人，以为算法界的领军人物）所说：”不成熟的优化是所有罪恶之源“，在优化之前我们要深思熟虑，全面地考虑改动所带来的影响。

选择排序Java实现

import java.util.Random;

public class Selection {
	public static void sort(Comparable[] arr) {
		for (int i = 0; i < arr.length; i++) {
			int min = i;
			for (int j = i+1; j < arr.length; j++) {
				if (less(arr[j], arr[min]))

3.插入排序

插入排序是初级排序算法中最重要的一个，因为它可以不仅可以作为更高级算法的一个步骤（希尔排序，可以将其理解为优化的插入排序），还可以对其他高级排序算法进行优化（因为对于小数组排序，插入排序速度很快，可将其作为优化快速排序的一个手段）。因此，插入排序的学习是十分重要的。

1. 基本原理

![poker](E:\Qisir\Pictures\Saved Pictures\poker.png)

通过一个简单的例子了解插入排序，比如说上述扑克牌为你的手牌，现在你摸到了一张A（也就是6的右侧是A），你该如何整理这几张牌呢？先将A与6比，A比6要小，故调换两者的顺序，依次下去直到A到合适的位置（在这里是最左侧）。（这是最初级的插入排序的理解，后面会有优化的方法）

插入排序的基本原理可以概括为”黑白分明“，这是对插入排序过程中数组元素特点的形象概括。插入排序通过将每个元素与之前的元素进行比较，若两者顺序错误则互换，并依次进行下去，直到其处于“正确”的位置。==需要注意的是，这个正确是相对的，也就是说这个元素位置现在确实正确，但排序继续进行下去它的位置可能会变，这与选择排序不同，选择排序的正确位置就是最终位置。== 由此造成的结果是，选择排序过程中数组被一分为二：有序（左侧）与未排序区（右侧），正所谓黑白分明。

2. 实现

插入排序伪代码：

InsertionSort:

Loop (i: From 1 To N-1): //这里注意一下是从1开始，前面的排序算法是从0开始，因为插入排序是从后往前比的

​ Loop (j: From i To 1):

​ if arr[j] < arr[j-1]:

​ 交换arr[j]与arr[j-1]

有的同学可能会发现，伪代码可能存在严重缺陷，如果已经交换好了，可是j仍是会继续减小直到0，这样岂不是产生了很大的浪费？我们可以像冒泡排序的不成熟改进一样引入一个bool变量来解决这个问题，但是这样改的话对于其性能的分析可能会带来麻烦（这个存在争议），而且这个改进带来的进步可能不值得这样做。

插入排序的java实现

package com.qisir.pku.algorithm.sort;

import java.util.Random;

public class Insertion {
	public static void sort(Comparable[] arr) {
		for (int i = 0; i < arr.length; i++) {
			for (int j = i; j > 0 && less(arr[j], arr[j-1]); j--) {
				exch(arr, j, j-1);
			}
		}
	}
	
	// 这样可能会起到一定的优化作用，但可能不值得这样做，因为收益太小
	public static void advancedSort(Comparable[] arr) {
		for (int i = 0; i < arr.length; i++) {
			boolean flag = true;
			for (int j = i; j > 0 && flag; j--) {
				if (!less(arr[j], arr[j-1]))
					flag = false;
				else
				    exch(arr, j, j-1);
			}
		}
	}
	
	// 比较两可比较对象，返回v是否小于w
	private static boolean less(Comparable v, Comparable w) {
		return v.compareTo(w) < 0;
	}
	
	//交换被排序数组中两个元素
	public static void exch(Comparable[] arr, int i, int j) {
		Comparable t = arr[i];
		arr[i] = arr[j];
		arr[j] = t;
	}
	
	//输出数组内容
	private static void show(Comparable[] arr) {
		for (int i = 0; i < arr.length; i++)
			System.out.print(arr[i]);
		System.out.println();
	}
	
	//检查数组是否已经按从小到大的顺序排列
	public static boolean isSorted(Comparable[] arr) {
		for (int i = 9; i < arr.length; i++)
			if (less(arr[i], arr[i-1])) return false;
		return true;
	}
	
    //简单的测试
	public static void main(String[] args) {
		Random random = new Random();
		Comparable[] arr = new Comparable[10];
		for (int i = 0; i < arr.length; i++)
			arr[i]  = random.nextInt(10);
		show(arr);
		advancedSort(arr);
		show(arr);
	}
}

4. 希尔排序

希尔排序是建立在插入排序上的，可以说是升级版的插入排序。其有效地解决了插入排序的缺点，是一个实现相对简单的高级排序算法，甚至有专家建议，如果你需要解决一个排序问题又没有系统排序函数可用，可以先用希尔排序，然后再考虑是否替换为更加复杂的算法。

1. 原理

插入排序的局限性在于其只能交换相邻元素，如果目标元素与其适当位置离得非常远，这样的效率会非常低，我们仍是得一个个交换元素。希尔排序则是在交换元素的方式上做了一些改变。希尔排序通过交换不相邻元素以对数组的局部进行排序，并最终用插入排序将局部有序数组排序。

希尔排序的核心思想是是数组中任意间隔为h的元素都是有序的，（当h较大时，能够实现相隔较远元素间的交换，h逐渐减小到1，当h为1时，数组自然就是有序的了。）可以概括为**“细致入微”**。

重要的概念

h有序数组： 数组中间隔为h的数组元素都是有序的。可以将其理解为是h个相互独立的的有序数组**“编织”**（注意这个编织）而成。

**递增序列：**我们在上面提到h会从某个值逐渐降低至1，其实h是有说道的。任何一个首项为1的递增序列都对应着一个h，序列的选取会影响到希尔排序的效率，不过这是专家的工作了。（具体的递增序列会在代码中体现）

2. 实现

希尔排序伪代码

ShellSort:

set h = 1

//选定合适的h: 首先把h升高到极限

while h < arr.length / 3:

​ *h = 3 h + 1 //这就是之间提到的递增序列之一，这是书上推荐的一个

// 排序的主要流程

while ( h >= 1):

​ Loop (i: From h To N):

​ Loop (j: From i DownTo h; By h):

​ if arr[j] < arr[j - h]:

​ 交换arr[j]与arr[j-h]

​ h = h/3

希尔排序Java实现

package com.qisir.pku.algorithm.sort;

import java.util.Random;

//希尔排序可以认为是插入排序的推广
public class Shell {
	public static void sort(Comparable[] arr) {
		int N = arr.length;
		int h = 1;
		while (h < N/3) h = 3*h + 1;
		while (h >= 1) {
			for (int i = h; i < N; i++) {
				for (int j = i; j >= h && less(arr[j], arr[j-h]); j -= h)
					exch(arr, j, j-h);
			}
			h /= 3;
		}
	}
	// 比较两可比较对象，返回v是否小于w
	private static boolean less(Comparable v, Comparable w) {
		return v.compareTo(w) < 0;
	}
	
	//交换被排序数组中两个元素
	private static void exch(Comparable[] arr, int i, int j) {
		Comparable t = arr[i];
		arr[i] = arr[j];
		arr[j] = t;
	}
	
	//输出数组内容p
	public static void show(Comparable[] arr) {
		for (int i = 0; i < arr.length; i++)
			System.out.print(arr[i]);
		System.out.println();
	}
	
	//检查数组是否已经按从小到大的顺序排列
	public static boolean isSorted(Comparable[] arr) {
		for (int i = 9; i < arr.length; i++)
			if (less(arr[i], arr[i-1])) return false;
		return true;
	}
	
	public static void main(String[] args) {
		Random random = new Random();
		Comparable[] arr = new Comparable[10];
		for (int i = 0; i < arr.length; i++)
			arr[i]  = random.nextInt(10);
		show(arr);
		sort(arr);
		show(arr);
	}
	
}

5. 归并排序

1. 基础概念

在这里会详细介绍两种归并排序：自顶向下的归并排序和自底向上的归并排序。归并排序，从它的名字上就知道这种排序方法与归并操作有关。

归并操作：归并指的是将两个有序数组归并成一个更大的有序数组。

这里需要注意：归并操作会生成一个更大的数组，这个数组可以是我们新建的数组，也可以是原数组。你可能会比较奇怪，更大的有序数组怎么可能是自身呢？其实归并操作可以发生在一个数组的两半，我们称之为原地归并，也就是在原地进行的归并操作（事实上我们借助了辅助数组）

原地归并的抽象方法java实现

//注意，这段代码只是展示了归并操作的原理，其需要其他代码配合使用（见完全实现的归并排序）
public static void merge(Comparable[] arr, int lo, int mid, int hi) {
	int i = lo, j = mid + 1;
		//先把数组复制到辅助数组里面，辅助数组的初始化在另一个方法里
	for (int k = lo; k <= hi; k++)
		aux[k] = arr[k]; 
		
	for (int k = lo; k <= hi; k++) {
		if (i > mid) arr[k] = aux[j++];
		else if (j > hi) arr[k] = aux[i++];
		else if (less(aux[j], aux[i])) arr[k] = aux[j++];
		else arr[k] = aux[i++];
	}
}

归并操作的核心就在原地归并上。

2. 原理

归并排序是通过
